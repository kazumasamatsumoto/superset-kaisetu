### 2.2 元の要望内容

**問題**:
- 多数のシリーズを持つチャート（特に時系列）で、ツールチップが巨大になりすぎる
- ツールチップがチャート全体を覆い隠してしまう
- フィルターを適用してもツールチップに0値として表示される

**提案された解決策**:
1. **ユーザーが関心のあるシリーズのみを表示する設定**
2. **特定の時点で0またはnull値のシリーズを省略する設定**

**コミュニティの反応**:
- 👍 20+ reactions（多数のユーザーが同じ問題に遭遇）
- 複数のコメントで「この機能が欲しい」という声

### 2.3 なぜ閉じられたのか

**理由**: Issue自体には明確な閉じられた理由が記載されていないが、以下の可能性が高い：

1. **長期間（7年以上）実装されなかった**
   - 2018年から2025年まで未解決
   - 技術的な難しさではなく、優先度の問題

2. **inactive（非アクティブ）ラベル**
   - 30日以上活動がないIssueは自動的に閉じられるポリシー
   - ボット（Stale bot）による自動クローズの可能性

3. **関連する部分的な解決策が実装された**
   - 後述のremoveNulls機能など
   - ただし、元の要望は完全には満たされていない

**重要**: 閉じられているが「解決済み」ではない（Won't Fix や Duplicate のラベルもない）

## 3. 関連するPRと実装

### 3.1 PR #21296: 0値を表示する修正（2022年）

**タイトル**: `fix(plugin-chart-echarts): show zero value in tooltip`

**URL**: https://github.com/apache/superset/pull/21296

**内容**:
- **逆の問題を修正**: 0値がツールチップに表示されなかったバグ
- EChartsプラグインで、truthyチェックにより0が除外されていた
- このPR後、0値は正しく表示されるようになった

**実装詳細**:
```typescript
// 修正前（バグ）
if (value) {  // 0はfalsyなので除外される
  // ツールチップに追加
}

// 修正後
if (typeof value === 'number') {  // 0もnumberなので含まれる
  // ツールチップに追加
}
```

**Issue #4462との関係**:
- **矛盾している**: Issue #4462は「0を非表示にしたい」、PR #21296は「0を表示したい」
- これは、異なるユースケースが存在することを示している：
  - ケースA: 0は有効なデータなので表示すべき
  - ケースB: 0はノイズなので非表示にすべき
- **結論**: 設定で制御できるべき

### 3.2 removeNulls機能（既存機能）

**場所**: `plugin-chart-echarts/src/utils/series.ts:267`

```typescript
export function extractSeries(
  data: DataRecord[],
  opts: {
    removeNulls?: boolean,  // ← この機能
    // ... その他のオプション
  } = {},
): [SeriesOption[], number[], number | undefined] {
  // ...
  .filter(obs => !removeNulls || (obs[0] !== null && obs[1] !== null))
  // ↑ removeNullsがtrueの場合、nullを含む行を除外
}
```

**動作**:
- データポイント自体からnullを削除
- ツールチップ表示前の段階で処理

**Issue #4462との違い**:
- removeNulls: **データレベル**でnullを削除（チャート描画から除外）
- Issue #4462の要望: **ツールチップレベル**で0/nullを非表示（データは残す、表示のみ制御）

**図解**:

```
データ: [100, null, 0, 200]

removeNulls = true の場合:
  → データから削除: [100, 0, 200]
  → チャート描画: [100, 0, 200]  ← nullが消える
  → ツールチップ: [100, 0, 200]

Issue #4462の要望:
  → データ保持: [100, null, 0, 200]
  → チャート描画: [100, null, 0, 200]  ← すべて描画
  → ツールチップ: [100, 200]  ← 0とnullのみ非表示
```

**重要な違い**: removeNullsはチャート全体に影響するが、Issue #4462はツールチップのみに影響

## 5. なぜ実装されないのか（分析）

### 5.1 技術的な難しさ

**難易度**: 低（前述の実装案の通り、簡単に実装可能）

**技術的障壁**: なし

### 5.2 優先度の問題

**考えられる理由**:

1. **ニッチなユースケース**
   - すべてのユーザーが必要とする機能ではない
   - 回避策が存在する（SQLで除外、itemモード使用）

2. **設計上の哲学的な問題**
   - 「0は有効なデータか、ノイズか？」という判断はユーザー依存
   - デフォルト設定が難しい
   - 混乱を招く可能性

3. **他の優先度の高い機能**
   - AI統合、パフォーマンス改善など、より広範なユーザーに影響する機能
   - リソースの制約

4. **コミュニティからのPR不足**
   - Issue #4462に対するPRが提出されていない
   - コミュニティ主導の開発では、誰かが実装して貢献する必要がある

### 5.3 設計上の課題

**議論されるべき点**:

1. **スコープの問題**
   - 0のみ非表示？ nullのみ？ 両方？
   - ユーザーが選択できるべきか？

2. **適用範囲の問題**
   - すべてのチャートタイプに適用？
   - 特定のチャート（Timeseries、Mixed）のみ？

3. **他の機能との相互作用**
   - `removeNulls`との関係は？
   - Stacked modeとの関係は？（MISSING_VALUES_REPORT.md参照）

4. **後方互換性**
   - デフォルトONかOFFか？
   - 既存のダッシュボードへの影響は？

#### オプションA: セルフ実装（推奨）

**手順**:
1. TOOLTIP_HYBRID_FEATURE.mdの実装案を適用
2. ローカル環境でテスト
3. 社内のSupersetインスタンスで使用

**メリット**:
- ✅ 即座に利用可能
- ✅ 自社のニーズに最適化

**デメリット**:
- ❌ Supersetのアップグレード時にマージ作業が必要
- ❌ 他のユーザーは利用できない

#### オプションB: コミュニティへの貢献

**手順**:
1. セルフ実装してテスト
2. GitHub Issueを再開（Issue #4462にコメント、または新しいIssueを作成）
3. Pull Requestを作成
4. コミュニティレビューを受ける
5. 本家にマージされる

**メリット**:
- ✅ アップグレード時のマージ不要（公式機能として組み込まれる）
- ✅ 他のユーザーも利用可能
- ✅ コミュニティへの貢献

**デメリット**:
- ❌ レビュープロセスに時間がかかる（数週間～数ヶ月）
- ❌ 設計変更を求められる可能性
- ❌ 最終的にマージされない可能性もある

**工数見積もり**:
- 実装: 1-2日
- テスト: 1日
- ドキュメント作成: 1日
- PRレビュー対応: 1-2週間（断続的）
- **合計**: 約2-3週間

#### オプションC: 別のアプローチ

**SQLレベルでの制御**:
```sql
-- 0値を除外
WHERE metric_value != 0

-- nullも除外
WHERE metric_value IS NOT NULL
```

**メリット**:
- ✅ コード変更不要
- ✅ 即座に利用可能
- ✅ パフォーマンス向上（データ転送量削減）

**デメリット**:
- ❌ 「値が0」と「データなし」の区別ができない
- ❌ すべてのクエリに手動で追加する必要がある
- ❌ チャートには0が表示されなくなる（ツールチップだけでなく全体に影響）
