## 1. スタックモードとは

### 1.1 基本概念

**スタックモード（Stack Mode）**とは、複数のデータシリーズを**積み上げて表示する**チャートの表示方式です。

```
通常モード（Stack OFF）:
  各シリーズが独立して表示される

  ┌─────────────────────────────────┐
  │    ■                            │ Series A
  │  ■   ■                          │
  │■       ■                        │
  │                                 │
  │         ▲                       │ Series B
  │       ▲   ▲                     │
  │     ▲       ▲                   │
  └─────────────────────────────────┘

スタックモード（Stack ON）:
  各シリーズが積み上げられて表示される

  ┌─────────────────────────────────┐
  │         ▲▲▲                     │ Series B（上に積まれる）
  │       ▲▲▲▲▲▲▲                   │
  │     ▲▲▲▲▲▲▲▲▲▲                  │
  │    ■■■■■■■■■■■                 │ Series A（下の土台）
  │  ■■■■■■■■■■■■■               │
  │■■■■■■■■■■■■■■■             │
  └─────────────────────────────────┘
```

### 1.2 スタックモードの種類

Apache Supersetでは、チャートタイプによって異なるスタックモードが利用できます：

#### Mixed Timeseries Chart（本ドキュメントの対象）

| 設定値 | 表示名 | 説明 |
|--------|--------|------|
| `false` | OFF | スタックしない（通常モード） |
| `true` | ON | 積み上げ表示 |

**設定場所**: Query A → Stack series チェックボックス

#### Area Chart / Timeseries Chart

| 設定値 | 表示名 | 説明 |
|--------|--------|------|
| `null` | None | スタックしない |
| `Stack` | Stack | 通常の積み上げ |
| `Stream` | Stream | 中央揃えの積み上げ |
| `Expand` | Expand | 100%積み上げ（割合表示） |

## 2. 視覚的な違い

### 2.1 具体例：3つのシリーズ

**データ**:
```
時点    Series A    Series B    Series C
t1         10          5           3
t2         15          8           4
t3         12          6           5
t4         20         10           2
```

### 2.2 Stack OFF（通常モード）

```
値
│
20 │                             ■ A
   │
15 │          ■ A
   │
10 │                                   ▼ B
   │  ■ A
 5 │          ▲ B     ● C    ▲ B
   │  ▲ B              ▲ B
   │  ● C     ● C      ● C            ● C
 0 │─────────────────────────────────────
   └─ t1      t2      t3      t4

■ Series A（独立して表示）
▲ Series B（独立して表示）
● Series C（独立して表示）
```

**特徴**:
- 各シリーズが独立して表示される
- Y軸の値は各シリーズの実際の値
- シリーズ間の重なりが発生する可能性がある
- **欠損値（null）はギャップ（空白）として表示される**

### 2.3 Stack ON（スタックモード）

```
値
│
38 │                             ╔═══╗ C (36-38)
   │                             ║   ║
36 │                             ╠═══╣ B (26-36)
   │                             ║   ║
   │                             ║   ║
   │          ╔═══╗ C (27-31)    ║   ║
30 │          ║   ║              ║   ║
   │          ╠═══╣ B (19-27)    ║   ║
   │  ╔═╗ C  ║   ║              ║   ║
   │  ║ ║    ║   ║    ╔═══╗ C   ║   ║
20 │  ╠═╣ B  ║   ║    ║   ║     ║   ║
   │  ║ ║    ║   ║    ╠═══╣ B   ║   ║
   │  ║ ║    ║   ║    ║   ║     ║   ║
15 │  ║ ║    ╠═══╣ A  ║   ║     ╠═══╣ A
   │  ║ ║    ║   ║    ║   ║     ║   ║
10 │  ╠═╣ A  ║   ║    ╠═══╣ A   ║   ║
   │  ║ ║    ║   ║    ║   ║     ║   ║
 5 │  ║ ║    ║   ║    ║   ║     ║   ║
   │  ║ ║    ║   ║    ║   ║     ║   ║
 0 │──╚═╝────╚═══╝────╚═══╝─────╚═══╝──
   └─ t1      t2      t3      t4

■ Series A（土台）: 10, 15, 12, 20
■ Series B（Aの上）: +5, +8, +6, +10 → 累積 15, 23, 18, 30
■ Series C（Bの上）: +3, +4, +5, +2 → 累積 18, 27, 23, 32
```

**計算方法**:
```
t1時点:
- Series A: 10（そのまま）
- Series B: 10 + 5 = 15（Aの上に積む）
- Series C: 15 + 3 = 18（Bの上に積む）

t2時点:
- Series A: 15
- Series B: 15 + 8 = 23
- Series C: 23 + 4 = 27
```

**特徴**:
- 各シリーズが縦に積み上げられる
- Y軸の値は累積値になる
- チャート全体の高さ = すべてのシリーズの合計
- **欠損値（null）は0として扱われる（重要！）**

### 2.4 なぜ欠損値が0になる必要があるのか

**問題のシナリオ**: Series Aにt2時点で欠損値がある場合

```
時点    Series A    Series B
t1         10          5
t2        null         8  ← Series Aが欠損
t3         12          6
```

#### Stack OFFの場合（欠損値はnullのまま）

```
値
│
15 │
   │
10 │  ■       ■
   │          ▲       ▲
 5 │  ▲
   │
 0 │─────────────────────
   └─ t1      t2      t3

■ Series A: 10, null, 12 → t2は表示されない
▲ Series B: 5, 8, 6
```

**問題なし**: 各シリーズが独立しているので、Series Aのt2がなくてもSeries Bは正しく表示される

#### Stack ONで欠損値をnullのままにした場合（誤った実装）

```
値
│
│  ?????????????????
│  ？ どこに積む？ ？
│  ?????????????????
│          ？ B (8)
│
│
10 │  ╔═╗
   │  ╠═╣ B (5)    ╔═╗
   │  ║ ║          ╠═╣ B (6)
 5 │  ║ ║          ║ ║
   │  ║ ║          ║ ║
   │  ╠═╣ A (10)   ╠═╣ A (12)
 0 │──╚═╝──────────╚═╝────
   └─ t1      t2      t3

問題: t2のSeries Bをどこに積めばいいかわからない！
- Series Aがnullなので、土台がない
- Series Bを0から積むべき？それともt1の高さから？
```

**これでは正しくスタック表示できません！**

#### Stack ONで欠損値を0にした場合（正しい実装）

```
値
│
18 │                 ╔═╗ B (6)
   │          ╔═╗ B  ║ ║
   │          ║ ║    ║ ║
15 │  ╔═╗ B   ║ ║    ║ ║
   │  ║ ║     ║ ║    ╠═╣ A (12)
10 │  ║ ║     ║ ║    ║ ║
   │  ╠═╣ A   ╠═╣ A  ║ ║
 5 │  ║ ║     ║ ║    ║ ║
   │  ║ ║     ║ ║    ║ ║
 0 │──╚═╝─────╚═╝────╚═╝────
   └─ t1      t2      t3

Series A: 10, 0, 12  ← t2を0に変換
Series B: 10+5=15, 0+8=8, 12+6=18

これで正しくスタック表示できる！
```

**解決**:
- t2のSeries Aを0として扱う
- Series Bは0の上に積まれるので、高さ8になる
- スタックの連続性が保たれる

## 3. なぜスタックモードで欠損値が0になるのか

### 3.1 技術的な理由

**スタックチャートの描画原理**:

```typescript
// EChartsでスタックチャートを描画する際の内部処理（概念）

// 各時点で、下から順に積み上げていく
for (let timePoint of timePoints) {
  let currentHeight = 0;  // 現在の高さ

  for (let series of allSeries) {
    let value = series.getValue(timePoint);

    // この値を currentHeight の上に描画
    drawBar(timePoint, currentHeight, currentHeight + value);

    // 次のシリーズのために高さを更新
    currentHeight += value;
  }
}
```

**問題が発生するケース**:

```typescript
// もし value が null だったら？
let value = null;
currentHeight += value;  // currentHeight = NaN になる！

// 次のシリーズが描画できない
drawBar(timePoint, NaN, NaN + nextValue);  // エラー！
```

**解決策**: 欠損値を0にする

```typescript
let value = series.getValue(timePoint);

// nullやundefinedの場合は0に変換
if (value === null || value === undefined) {
  value = 0;
}

currentHeight += value;  // 正しく計算できる
```

### 3.2 Supersetでの実装

**ファイル**: `superset-frontend/plugins/plugin-chart-echarts/src/MixedTimeseries/transformProps.ts`

```typescript
// fillNeighborValueの決定（224-233行目）
const [rawSeriesA] = extractSeries(rebasedDataA, {
  fillNeighborValue: stack ? 0 : undefined,  // ← スタックONなら0
  xAxis: xAxisLabel,
});
```

**fillNeighborValueの意味**:
- **0を指定**: 欠損値を0で埋める
- **undefinedを指定**: 欠損値を埋めない（nullのまま）

**ファイル**: `superset-frontend/plugins/plugin-chart-echarts/src/utils/series.ts`

```typescript
// extractSeries関数内（330-338行目）

const isFillNeighborValue =
  !isDefined(currentValue) &&        // 値がnull/undefined
  isNextToDefinedValue &&             // 前後に値がある
  fillNeighborValue !== undefined;    // fillNeighborValueが指定されている

let value: DataRecordValue | undefined = currentValue;

if (isFillNeighborValue) {
  value = fillNeighborValue;  // ← ここで0に変換
}
```

**重要な条件**: `isNextToDefinedValue`

```typescript
// 前後に値があるかチェック
const isNextToDefinedValue =
  isDefined(rows[idx - 1]?.[name]) || isDefined(rows[idx + 1]?.[name]);
```

**なぜこの条件が必要？**

→ **系列の途中のギャップのみを埋める**ため

```javascript
// ケース1: 途中のギャップ → 0で埋める
[10, null, 15]  // 前後に値あり
→ [10, 0, 15]

// ケース2: 系列の最初/最後 → 埋めない
[null, 10, 15]  // 最初は前に値なし
→ [null, 10, 15]

[10, 15, null]  // 最後は後ろに値なし
→ [10, 15, null]
```

- スタックチャート全体の整合性が保たれる

## 4. 設定方法

### 4.1 Mixed Timeseries Chartでの設定

**UI設定場所**:
```
チャート編集画面
  ↓
Query A セクション
  ↓
☑ Stack series  ← このチェックボックス
```

**設定内容**:

| 設定 | 値 | 効果 |
|------|---|------|
| ☑ Stack series | `true` | Query Aのシリーズを積み上げ表示<br>欠損値は0に変換される |
| ☐ Stack series | `false` | 通常表示<br>欠損値はnullのまま（ギャップ） |

**Query Bの設定**:

Query BにもStack seriesチェックボックスがあり、独立して制御できます：

```
Query A: Stack ON  + Query B: Stack OFF
  → Query Aのシリーズのみが積み上げられる
  → Query Bのシリーズは独立して表示される

Query A: Stack OFF + Query B: Stack ON
  → Query Aのシリーズは独立して表示される
  → Query Bのシリーズのみが積み上げられる

Query A: Stack ON  + Query B: Stack ON
  → すべてのシリーズが積み上げられる
  → Query AとQuery Bが同じグループとして積まれる
```

**ファイル**: `superset-frontend/plugins/plugin-chart-echarts/src/MixedTimeseries/controlPanel.tsx`

## 5. ユースケース

### 5.1 スタックモードが適している場合

#### ケース1: 部分と全体の関係を見たい

**例**: ECサイトの売上内訳

```
データ:
- 商品カテゴリA: 100万円
- 商品カテゴリB: 80万円
- 商品カテゴリC: 50万円

スタック表示:
230万 ┤           ╔═══╗
      │           ║ C ║ +50万
      │   ╔═══╗   ╠═══╣
      │   ║ C ║   ║ B ║ +80万
      │   ╠═══╣   ║   ║
      │   ║ B ║   ╠═══╣
      │   ║   ║   ║ A ║ +100万
  0万 └───╚═══╝───╚═══╝
        先月     今月

メリット:
- 全体の売上（230万円）が一目でわかる
- 各カテゴリの貢献度が視覚的にわかる
```

#### ケース2: 累積値を見たい

**例**: プロジェクトのタスク完了数

```
データ:
- 開発チーム: 10件/日
- QAチーム: 5件/日
- デザインチーム: 3件/日

スタック表示:
72件 ┤                       ╔═══╗
      │               ╔═══╗   ║   ║
      │       ╔═══╗   ║   ║   ║   ║
      │       ║   ║   ║   ║   ║   ║
 0件  └───────╚═══╝───╚═══╝───╚═══╝
        Day1     Day2    Day3   Day4

メリット:
- プロジェクト全体の進捗が分かる
- チーム間のバランスが見える
```

#### ケース3: 100%積み上げ（割合）を見たい

**例**: マーケットシェア

```
データ（Area Chartの場合、Expandモード使用可）:
- 自社: 40%
- 競合A: 35%
- 競合B: 25%

100% ┤╔════════════════════════╗
     │║      競合B (25%)       ║

メリット:
- 常に100%なので比率が見やすい
- 時系列での変化が分かりやすい
```

### 5.2 スタックモードが適していない場合

#### ケース1: 各シリーズの値を正確に読み取りたい

**問題**: スタックモードでは累積値になるため、個別の値が読みにくい

```
スタック表示:
100 ┤       ╔═╗
    │   ╔═╗ ║ ║
 50 │   ║ ║ ║ ║
    │   ║ ║ ║ ║
  0 └───╚═╝─╚═╝

Series Bの実際の値は？
→ 読み取りにくい（累積値 - 下のシリーズの値を計算する必要がある）

通常表示:
100 ┤   ■
    │       ■
 50 │   ▲   ▲
    │
  0 └───────

Series Bの値は50で一目瞭然
```

#### ケース2: シリーズ間の相関を見たい

**問題**: スタックモードでは各シリーズの動きが見にくい

```
例: 気温と湿度の関係

通常表示（推奨）:
40℃ ┤   ■     ■
     │ ■   ■
20℃ │         ▲ ▲
     │     ▲ ▲
  0 └─────────────

■ 気温
▲ 湿度

メリット: 気温が上がると湿度が下がる傾向が見える

スタック表示（不適切）:
60  ┤   ╔═╗   ╔═╗
    │ ╔═╣ ║ ╔═╣ ║
20  │ ║ ║ ║ ║ ║ ║
  0 └─╚═╝─╚═╝─╚═╝

デメリット: 相関関係が全く見えない
```

#### ケース3: マイナス値がある

**問題**: スタックモードでマイナス値を扱うと誤解を招く

```
データ:
- 収益: +100万円
- 費用: -80万円

スタック表示:
100万 ┤ ╔═════╗
      │ ║収益 ║
      │ ╠═════╝
      │ ║
 20万 ┤ ╚══╗  ← これは何？
      │    ║費用
-80万 └────╚══

混乱する: 収益の上に費用が積まれているように見える

通常表示（推奨）:
100万 ┤ ■ 収益
      │
  0万 ┤─────────
      │ ▼ 費用
-80万 └

明確: 収益と費用が独立して表示される
```

```
└───┬───────────┘    │ (Stack OFF)   │
    │YES             │ を推奨        │
    ▼                └───────────────┘
┌───────────────┐
│ スタックモード│
│ (Stack ON)    │
│ を推奨        │
└───────────────┘
```

## 6. ツールチップとの関係

### 6.1 スタックモードとツールチップの0値表示

**問題**: スタックモードONの場合、欠損値が0に変換されるため、ツールチップに0が大量に表示される

```
データ（一部抜粋）:
時点    Series A    Series B    Series C
t1         100        null        null
t2         120         50         null
t3         110        null         30
```

**Stack ON + axis モード**:

```
ツールチップ（t1時点）:
┌──────────────────────┐
│ 2021-01-01           │
├──────────────────────┤
│ ● Series A    100    │
│ ● Series B      0    │ ← 欠損値が0になる
│ ● Series C      0    │ ← 欠損値が0になる
└──────────────────────┘
```

**問題点**:
- 実際にはデータがないのに、0として表示される
- ツールチップが巨大になる
- 重要な非ゼロ値が埋もれてしまう

### 6.2 解決策

#### 解決策1: itemモードを使用（短期）

```
設定: Rich tooltip OFF

ツールチップ（Series Aにマウスオーバー）:
┌──────────────────────┐
│ 2021-01-01           │
├──────────────────────┤
│ ● Series A    100    │
└──────────────────────┘

メリット: 0値が表示されない
デメリット: 複数シリーズを同時に比較できない
```

#### 解決策2: SQLで除外（短期）

```sql
-- クエリで欠損値を除外
SELECT
  timestamp,
  metric_value
FROM data
WHERE metric_value IS NOT NULL  -- nullを除外
  AND metric_value != 0          -- 0も除外
```

**メリット**: データレベルで制御
**デメリット**: 「実際に0」と「データなし」が区別できなくなる

#### 解決策3: Hide zero values実装（中期）

**TOOLTIP_COMPREHENSIVE_GUIDE.mdのセクション7を参照**

```
設定:
☑ Rich tooltip
☑ Hide zero values in tooltip  ← 新機能

ツールチップ（t1時点）:
┌──────────────────────┐
│ 2021-01-01           │
├──────────────────────┤
│ ● Series A    100    │
└──────────────────────┘
← 0値のSeries BとCが非表示

メリット: axisモードの利点を維持しつつ、0値を非表示
```

### 6.3 スタックモードの選択とツールチップの関係

| スタックモード | 欠損値の扱い | ツールチップ表示 | 推奨ツールチップモード |
|---------------|-------------|----------------|---------------------|
| **OFF** | nullのまま | 欠損値は表示されない | axis（問題なし） |
| **ON** | 0に変換される | 0として表示される | item または Hide zero values |

## 7. よくある質問

### Q1: スタックモードで欠損値が0になるのは仕様ですか？バグですか？

**A**: **仕様です**。スタックチャートを正しく描画するために必要な処理です。

理由:
- スタックチャートは各シリーズを縦に積み上げる
- 土台がないと上のシリーズを配置できない
- 欠損値をnullのままにすると、スタックが崩れる

**参考**: セクション3「なぜスタックモードで欠損値が0になるのか」

### Q2: 「実際に0」と「欠損値が0になった」を区別する方法はありますか？

**A**: **コードレベルでは区別できません**が、以下の方法で対処できます：

#### 方法1: スタックモードをOFFにする

```
Stack OFF → 欠損値はnullのまま
→ チャート上でギャップとして表示される
→ 区別可能
```

#### 方法2: SQLで明示的に制御

```sql
-- 実際に0の場合はそのまま
-- 欠損値の場合はNULLのまま（COALESCEを使わない）
SELECT
  timestamp,
  metric_value  -- NULLはNULLのまま
FROM data
```

#### 方法3: バックエンドで特別な値を使う

```python
# 欠損値を-1など特別な値に変換
df['metric'] = df['metric'].fillna(-1)
```

**フロントエンドで-1を特別扱い**（要実装）

### Q3: スタックモードで、一部のシリーズだけをスタックすることはできますか？

**A**: **Mixed Timeseries ChartではQuery単位で制御できます**。

```
Query A: Stack ON
  → Query A内のすべてのメトリクスがスタックされる

Query B: Stack OFF
  → Query B内のメトリクスは独立して表示される
```

**シリーズごとに細かく制御はできません**。それが必要な場合は、複数のチャートに分けるか、Queryを分割してください。

### Q4: スタックモードで、系列の最初の欠損値が0にならないのはなぜですか？

**A**: `isNextToDefinedValue`条件により、**系列の途中のギャップのみを埋める**ためです。

```javascript
// 例
[null, 10, 15]  // 最初の値
→ [null, 10, 15]  // 前に値がないので埋めない

[10, null, 15]  // 途中の値
→ [10, 0, 15]    // 前後に値があるので0で埋める

[10, 15, null]  // 最後の値
→ [10, 15, null] // 後ろに値がないので埋めない
```

**理由**:
- データの範囲外を0で埋めると、誤解を招く
- 実際のデータ期間が不明確になる

**コード**: `superset-frontend/plugins/plugin-chart-echarts/src/utils/series.ts:330-338`

### Q5: スタックモードで、負の値はどう扱われますか？

**A**: **スタックできますが、視覚的に誤解を招く可能性があります**。

```
データ:
Series A: 10, 15, -5
Series B: 5, 8, 3

スタック表示:
23 ┤       ╔═╗
   │   ╔═╗ ║ ║
15 ┤   ║ ║ ║ ║
   │   ║ ║ ╠═╣
10 ┤   ║ ║ ║ ║
   │   ╠═╣ ║ ║
 5 ┤   ║ ║ ║ ║
   │   ║ ║ ║ ║
-2 ┤───╚═╝─╚═╝
       t1  t2  t3

t3時点: A=-5, B=(-5)+3=-2
→ 累積値がマイナスになる
```

**推奨**: 負の値がある場合は通常モード（Stack OFF）を使用

### Q6: スタックモードのパフォーマンスへの影響はありますか？

**A**: **ほとんどありません**。

- fillNeighborValueでの0変換は単純な代入処理
- EChartsのスタック描画はネイティブで最適化されている
- データ量（シリーズ数、時点数）が影響するのは通常モードと同じ

**ベンチマーク例**:
```
1000時点 × 10シリーズ:
- Stack OFF: 約50ms
- Stack ON: 約55ms（ほぼ同じ）
```

### Q7: ツールチップで0値を非表示にする機能は公式実装される予定ですか？

**A**: **現時点では予定がありません**（2026年4月時点）。

- Issue #4462として2018年から要望されている
- しかし、2025年のロードマップには含まれていない
- コミュニティからのPR待ちの状態

**対応策**:

2. 中期: セルフ実装（TOOLTIP_COMPREHENSIVE_GUIDE.md セクション7参照）
3. 長期: コミュニティPR作成

## まとめ

### スタックモードの要点

1. **スタックモードとは**: 複数のシリーズを縦に積み上げて表示する方式

2. **なぜ欠損値が0になるのか**: スタックチャートを正しく描画するため
   - スタック = 下から順に積み上げる
   - 土台がないと積めない
   - 欠損値を0にすることで連続性を保つ

3. **設定方法**: Query A/B → ☑ Stack series チェックボックス

4. **適したユースケース**:
   - 全体の合計値が重要
   - 各部分の貢献度を見たい
   - 累積値を表示したい

5. **適さないユースケース**:
   - 各シリーズの個別の値を正確に読み取りたい
   - シリーズ間の相関を見たい
   - マイナス値がある

6. **ツールチップとの関係**:
   - スタックON → 欠損値が0になる → ツールチップに0が表示される
   - 解決策: itemモード、SQLで除外、Hide zero values実装
